- Gateway device runs `code/service/` Node.js service

## 4. Quality Assurance Procedures

### 4.1 Physics Check (ρgQH formula)
Every reading is checked against the theoretical maximum generation:
```
EG_max = ρ × g × Q × H × η × Δt
       = 997 kg/m³ × 9.81 m/s² × Q × H × η × Δt
```
If `generatedKwh` exceeds `EG_max × 1.05` (5% tolerance), the reading
is flagged or rejected depending on trust score.

### 4.2 Temporal Consistency Check
- Gap detection: readings must arrive within configurable window (default: 2h)
- Spike detection: generation change > 50% between consecutive readings triggers flag
- Timestamp validation: future timestamps and clock drift > 5 min are rejected

### 4.3 Environmental Bounds Check
All sensors must be within ACM0002-acceptable ranges:

| Sensor | Perfect range | Acceptable | Questionable |
|--------|--------------|------------|-------------|
| pH | 6.5 – 8.5 | 6.0 – 9.0 | 5.5 – 9.5 |
| Turbidity | 0 – 50 NTU | 0 – 100 NTU | 0 – 200 NTU |
| Temperature | 0 – 30°C | -5 – 35°C | -10 – 40°C |

### 4.4 Statistical Anomaly Detection
Z-score analysis against rolling 30-reading window:

| Z-score | Classification | Action |
|---------|---------------|--------|
| < 1.0 | Normal | APPROVED |
| 1.0 – 2.0 | Acceptable | APPROVED with note |
| 2.0 – 2.5 | Questionable | FLAGGED |
| 2.5 – 3.0 | Suspicious | FLAGGED for review |
| > 3.0 | Anomalous | REJECTED |

### 4.5 AI Trust Scoring
Weighted composite score from all checks:

```
Trust = (physics × 0.40) + (temporal × 0.25) + (environmental × 0.20) + (statistical × 0.15)
```

| Trust score | Outcome |
|------------|--------|
| ≥ 0.90 | APPROVED — eligible for REC minting |
| 0.70 – 0.90 | FLAGGED — manual review required |
| < 0.70 | REJECTED — excluded from ER calculation |

## 7. Non-Conformance Procedure

1. Reading rejected → logged to attestation store with rejection reason
2. If rejection rate > 5% in a period → monitoring report flagged for VVB review
3. If physics check fails consistently → site inspection triggered
4. All non-conformances recorded on-chain via HCS message type `MRV_NONCONFORMANCE`
